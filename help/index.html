I know the help is virtualy non-existent.

But over the coming weeks, I will be updating the repo with all of the html help pages.

Note: The Help section will be just simple HTML.
I did this to enable the user to be able to annotate the help section for thier own purpose, make notes, reminders, etc...

The notes you make should be 'seperate' html pages than the actual help pages. (a prefix maybe?)
Link all of them together with some sort of index page, same as this page, is an index page, with links to all the help pages.
Now, you that have a 'top' or 'index' html page for your notes-help, just add a link to that page, here on the main index help page.

Because this entire help directory will change over time, as I add all the help content.
So, make sure you 'prefix' all of your content or something like that so it will not get orver-written when updating the help.

Updating the help will be, simply replacing all files in this folder, with all files form the help directory in the repo.
Over-writing and therefore updating some pages, while adding new ones, same with any images as well.
After that, you add your index page's 'one-link' back to 'thi's main help 'index' page.
Fin. All updated :)
